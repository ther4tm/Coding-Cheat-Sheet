HTML Cheat Sheet

Below is a list of all HTML tags and a brief description of what they do and any notes on correct syntax.

Main Root

Tag Name Description
<html> HTML This is the top level element in any webpage. All other items must be contained within this tag. This is referred to as the root element.

Document Metadata

The metadata of a page is best thought of as invisible information that is used to help browsers, search engines and software in general read and render the other elements of a page.

Tag Name Description
<!--, --> Comment Text placed between these tags is treated as a comment. It is a way for notes to be added to the code that does not affect the page in any way.
<head> Head This contains invisible data that helps the browser to determine certain things like the title of the page or where to find the linked CSS stylesheet.
<link> Link This specifies relationships between the current document and an external resource. For example this can provide a link to a CSS stylesheet that will then style the page.
<style> Style This allows for writing a section of CSS that is contained within this document and not a linked external stylesheet. This is only applied to the page it is contained within.
<title> Title This defines the document's title that is shown in the browser titles bar or a page's tab. This tag can only contain text.

Sectioning Root

Tag Name Description
<body> Body This represents the content of a HTML document. There is only ever one of these in a HTML page. Usually the <head> element is located above this in the organisation of the document.

Content Sectioning

Content sectioning elements allow for logical organisation of the page.

Tag Name Description
<article> Article This is a self contained composition in a document, page, application or site which is intended to be independently distributable or reusable. i.e. forum posts, blog entry or a user submitted comment.
<aside> Aside This is a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.
<footer> Footer This element typically contains information that is supplemental to an <article> or the <body> etc. It will contain things like information about the author of a post or copyright data or links to related document. These can be used under sectioning content or the section root element (<body>), you can have multiples of this in one page.
<header> Header This is similar to the <footer> in that you can have multiple instances of this within the document. It acts as introductory content , typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.
<h1>, <h2>, <h3>, <h4>, <h5>, <h6> Section Headings These are used to provide headings and titles to sections, they help to organise a site and bring structureda hierarchy. They can be reused is similar to the <footer> in that you can have multiple instances of this within the document. It acts as introductory content , typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.
<nav> Navigation This defines a section of the page where you can find navigational links. These links can be to other sections of the same page or different pages.

Text content

Text content elements are used to organise blocks and section of content within the <body> tags. They are important features to use correctly for accessibility and SEO purposes. They help to identify the purpose or structure of the content.

Tag Name Description
<blockquote> Blockquote This is the overall wrapper element for the entire table.
<div> Division Element This is the generic containing element for flow content. It has no effect on the content or layout until styled in some way using CSS.
<figure> Figure This represents self-contained content, usually an image and potentially with an optional caption, which is specified using the <figcaption> element. The figure, its caption, and its contents are referenced as a single unit.
<figcaption> Figure Caption This is a text element that is used to help describe the related figure.
<ul> Unordered List This is the container element for unordered lists. This requires <li> elements to provide content. When populated with content, text is displayed indented and in a bullet point fashion.
<ol> Ordered List This is the container element for ordered lists. This also requires <li> elements to provide content. When populated with content, text is displayed indented and in a numbered fashion.
<p> Paragraph This is the most basic way of displaying text. This is not confined to displaying text however as it is a form of page organisation it can also display images and forms.

Inline Text Semantics

These HTML elements can be used to alter a piece of text to give it styling properties or further structure and meaning. They are used inline to alter specific elements that may not fit the broader rules defined in the CSS.

Tag Name Description
<a> Link Element Together with its href attribute, this creates a hyperlink to web pages, files, email addresses, locations within the current page, or anything else a URL can address. Adding a title attribute to the link will allow tooltip's to be displayed when the link is briefly hovered over.
<b> Boldface This is an outdated tag that used to be the primary way of creating bold typefaced text. There are alternative ways to do this using CSS. It was also previously used to give importance to a word, again this is outdated and the <strong> tags should be used instead.
<br> Link Break This produces a line break in a section of text.
<cite> Citation This is used to mark a cited piece of work. This is often abbreviated in a context appropriate way.
<em> Emphasis This is used to place emphasis on a specific piece of text. This element can be nested with each level receiving greater degrees of emphasis.
<i> Italics This is used to differentiate text. Historically it put the text in italics and was used to signify various technical or other taxonimical designations.
<q> Quotation This is used to create quotations that do not require line breaks. Browsers by default render these with quotation marks around them.
<s> Strikethrough This places a strike through enclosed text.
<small> Small Text This transforms the contained text to one size smaller from the default or applied size. For example small text is transformed into x-small text.
<span> Span This tag creates an inline container element to which effects can be applied. Attributes such as id and class can help to bring further specificity to the enclosed text by creating unique version of the span tag.
<strong> Strong These tags provide emphasis on its contained content indicating a strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
<sub> Subscript These tags specifies inline text which should be displayed as subscript for solely typographical reasons. Subscripts are typically rendered with a lowered baseline using smaller text.
<sup> Superscript Specifies inline text which is to be displayed as superscript for solely typographical reasons. Superscripts are usually rendered with a raised baseline using smaller text.
<u> Underline Represents a span of inline text which should be rendered in a way that indicates that it has a non-textual annotation. This is rendered by default as a simple solid underline but may be altered using CSS.

Image and Multimedia

These tags help to populate the page with media other than text.

Tag Name Description
<audio> Audio This tag allows for embedding audio into the page.
<img> Image This tag allows for embedding images into the page.
<video> Video This tag allows for embedding video into the page.

Embedded Content

In addition to images and multimedia there are other types of content that can be embedded into the page.

Tag Name Description
<embed> Embed This embeds external content into the document. This content is usually provided by an external application or other source of interactive content such as a browser plug-in.

Scripting

Scripting allows for more dynamic content to be created on web pages.

Tag Name Description
<script> Script This is typically used to embed or refer to JavaScript code.
An example of typical syntax is;
<script src="main.js">
The default behaviour in browsers is for the script to be downloaded and executed in page order. So for example if the script is in the middle of the page, the script will be downloaded and executed before the bottom section of the page is downloaded and displayed. This behaviour can be modified with the attributes described below.

The defer attribute can be included within the tag to ensure that the entire HTML file has been parsed before executing the script. This allows for a quicker loading of the page.

The async attribute can be used within the tag to allow for the script to be downloaded in the background and executed once fully downloaded. This does not wait for the entire HTML to be parsed before executing.

Table elements

These tags are for the creation of tables within the page.

Tag Name Description
<table> Table This is the overall wrapper element for the entire table.
<thead> Table Head This is the wrapper element for the head of the table.
<tbody> Table Body This is the wrapper element for the main content of the table.
<tfoot> Table Foot This is the wrapper element for the footer content of the table.
<th> Table Header This is the wrapper element for rows within the table head. This is used within the <thead> tags to create rows for data within the head of the table. These usually serve to define the titles of each column.
<tr> Table Row This is the wrapper element for rows within the main body of the table. This is used within the <tbody> and <tfoot> tags to create rows for data.
<td> Table Data This is the wrapper element that contains individual cell data. The attributes colspan and rowspan can be used to help format the table within the HTML allowing for cells to stretch further than the default single cell layout. For example colspan="2" will allow for the defined cell to stretch over 2 column spaces rather than the default single cell layout. These can be used to help format the titular <th> and <tr> elements of the table too.

Forms

These tags are focused on the creation of forms and user interactivity. These tags usually provide users with the ability to create data and submit information for various purposes.

Tag Name Description
<form> Form This is the enabling element of user input on a site. Within the form tags it is necessary to include the attributes action and method.

The action attribute tells the browser where to send the form's information upon being submitted.
The method attribute tells the browser how the information is sent.

An example of correct syntax is shown below;
<form action="./submission.html" method="POST">

</form>
<label> Label The label tag allows for text, or a label, describing the input parameter.
The label tag should contain a for="" attribute. The purpose of this is to assign the label to an input.
The for="" of the label should match the id="" of the <input>

An example of syntax is shown below;
<label for="text">Text</label>
<input> Input The input tag allows for a number of different input methods to be defined allowing the user to write, choose and generally interact with the website.
The type attribute is the key to the interactivity provided by the <input> tag, these are listed individually below with their various settings.

An example of the typical syntax for an <input> tag is shown below, in this example we specify the inpute type as text;
<input type="text" name="name" id="id" >
Notice that there is no requirement for a closing tag or / element within the tags.
Also notice that the name="" attribute is what will be paired with the input when it is submitted by the user.
type="text" The input tag allows for a number of different input methods to be defined allowing the user to write, choose and generally interact with the website.
The type attribute is the key to the interactivity provided by the <input> tag, these are listed below with their various settings.
type="password" This attribute words in much the same way as the type="text" however when a user is writing something into the textbox it will display dots or asterisk's instead of the characters so nobody else looking at the screen can see the input.
type="number" This allows for the input of numbers. It will not accept non numerical input, acting as a quick front end validation tool.

This attribute also allows for additional functionality to be used within the <input> tag.
A min="1" and max="10" attribute can be used to ensure that the user input is kept within the defined range, in this case between 1 and 10.

A further attribute that can be used with this is the step="1" property. When a number input field is defined it will enter with an increase and decrease arrow button within the input field that will allow a user to increase or decrease the number. The step defines how much this arrow increases or decreases the number. So for example if we want the arrows to increase/decrease by 10 each time it is pressed we would define step="10"
type="range" This provides a slide bar that allows for selection from a pre-determined range of numbers.
This also requires additional attributes to be defined within the <input> tag.
A min="1" and max="10" attribute is required to provide the given range for a user to select from.
A further value="5" property is provided to give the slider a starting point.

An optional step="1" property can be provided to give the slider pre-determined increment values.
type="checkbox" The checkbox type allows for multiple choice input to be created within a form.
It is important to include a value="" property with each individual checkbox so that when the box is checked and submitted the value associated with that check will be returned by the form submission.

An example of syntax is shown below, in this example we have made a number of checkboxes for pizza toppings;

<input type="checkbox" name="topping" id="onion" value="onion">
<input type="checkbox" name="topping" id="mushroom" value="mushroom">

Note that each checkbox will have the same name property so when the form is submitted they will all be returned under the 'topping' name, the id and value properties are related to whatever the checkbox option is so the appropriate label is assigned and the correct value is returned.
type="radio" This provides a radio button that is usually used for boolean type selections (yes or no).
As with the checkbox attribute, it is important to include a value="" property with each individual radio button so that when the button is selected and submitted the value associated with that button will be returned by the form submission.

An example of syntax is shown below;

<input type="radio" name="name" id="yes" value="yes">
<input type="radio" name="name" id="no" value="no">
type="submit" This creates a submission button.
Adding a value="" attribute will create the button with whatever text is contained within parentheses of value
<select> & <option> Dropdown list The <select> tag creates the container for a dropdown list.
Much in the same way as other input tags it also requires a name="" and id="".

The <option> tags contain the items to populate the dropdown list.
These also require value=""
An example of correct syntax is shown below;

<select name="bun" id="bun">
<option value="sesame">Sesame</option>
<option value="brioche">Brioche</option>
</select>
type="text", list="", <datalist> & <option> Custom/searchable list The <datalist> tag allows for custom text to be inputted but also acts as a searchable list that contains pre-determined options.
Much in the same way as other input tags it also requires an id="" to be paired with the <input>.

The <option> tags contain the items to populate the searchable list.
These also require value=""

An example of correct syntax is shown below, in this example we create a searchable list of sauces with a few pre-determined options;

<input type="text" list="sauces" id="sauce" name="sauce">
<datalist id="sauces">
<option value="ketchup">Ketchup</option>
<option value="mayo">Mayo</option>
</datalist>
<textarea> Text Box The <textarea> tag allows for custom text to be submitted. This can be for things like comments, or custom messages that aren't covered by other provided options in a form.
This requires a name="" and id="" but it also requires attributes telling the browser how big to render the textbox.
The cols="" property tells the browser how wide the box should be and the rows="" property tells the broswer how tall it should be.
placeholder="" This creates a temporary piece of text that populates the textbox this attribute has been deeclared on.
When the user starts to type something the placeholder text will disappear.
This is usually to help direct the user in terms of expected input.
<button> Button This is an interactive element that can used to submit information or to provide an answer to a questionnaire.

Form Validation

These tags and attributes are focused on the validation of user input.

Tag Name Description
required Required This attribute is an additional keyword that can be inserted into <input> tags to ensure quick client-side validation on a form.
This will not allow the form to be submitted unless the associated <input> has been filled with some content.
minlength="" Minimum Length This attribute ensures that a user input reaches a set minimum length before being submitted.
maxlength="" Maximum Length This attribute ensures that a user input does not exceed a set maximum length before being submitted.
pattern="" Pattern This attribute ensures that a user input matches a pre-determined pattern.

Accessibility

These tags and attributes are focused on improving the accessibility of a website.

Tag Name Description
role="" Role This attribute is an additional attribute contained within HTML tags to provide context to people using screen readers as to the role of the content.

For example the role="presentation" attribute essential hides non essential parts of the site from the screen reader to allow for a less cumbersome expereince for the user. You would typically use this on containing divs or <ol> elements.
Note that this particular role keyword will hide an element and its necessary child elements so for example the <ol> AND <li> will be skipped from being read by a screen and instead it will skip straight to the text contained between the <li> tags.

There are many role keywords, see the W3 Website for more info.
Aria Properties These attributes are additional options that can be included within HTML tags to provide further context to people using screen readers as to the role and relationship of the content.

For example when a label is provided underneath an image as note on an artist, for someone using a screen reader it might not be clear what the paragraph below the image means. The property aria-label gives the screen reader additional information to read out loud to the user.

There are many role keywords, see the MDN Website for more info.
