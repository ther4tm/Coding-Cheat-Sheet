CSS Cheat Sheet

Below is a brief list of fundamental CSS tags and a brief description of what they do and any notes on correct syntax.

Syntax conventions

This section describes tags that are used as tools within the CSS.

Tag Name Description
/*, */ Comment Text placed between these tags is treated as a comment. It is a way for notes to be added to the code that does not affect the page in any way.

Selectors

To enable attributes to be applied to specific elements the relevant parts of the code need to be targetted. CSS has a number of ways to select specific items within the code through a combination of the methods below.

Tag Name Description
* Universal Selector This is the most wide ranging method of selection. It is essentially the select all parameter. This is usually at the start of the stylesheet as the specificity of the code becomes more complex the further you go into the document.
elementname Type Selector This targets HTML type tags. For example p would target all paragraph (<p>) tags.
.classname Class Selector This targets classes that have been defined within the HTML code. Classes are sections of code that are likely to be used multiple times which may require a blanket rule set that differs from the root level element. They can be defined as a supplemental attribute to an element. This allows a greater level of specificity when targetting elements within the page. For example as above, p would target all paragraph (<p>) tags however, by writing a class based rule you can further define additional attributes or override the original rules.
#idname ID Selector This targets elements that have been further defined with an id attribute. An id is intended to be used in a more sporadic and less frequent way than classes. They can be nested within a class to give further specificity when writing rules.
*[attr=value] Attribute Selector This targets very specific elements within the code. The syntax is written so that it selects a specific value of a specific attribute of the preceeding element and applies the rules to that item. Within this there is syntax to further specify how and when the attributes are relevant. For example a[href="https://example.org"] will select and apply rules to all links that contain "https://example.org".

Visual Rules

These attributes define how elements of HTML are displayed graphically such as the typeface, the colours and font weights.

Tag Name Description
font-family Font Family This defines the typeface to be used for the selected element. This can often contain multiple families written in series, this is done as a fallback option in case the browser doesn't have access to the first choice, it then defaults to the next choice.
font-size Font Size This defines the sized of the font to be used for the selected element. This can be defined in many different ways for example keywords can be used such as medium or small but also percentages or length based sizes can be given such as 80% or 12px.
font-weight Font Weight This defines the weight of the typeface to be used for the selected element. This can be defined using keywords such as bold, it can also be defined using numbers such as 600. The default values are medium and 400.
text-align Text Align This defines how the text of the selected element will be aligned. This defined as either left, center or right.
color Text Colour This defines the colour of the text to be used for the selected element. This can be defined in numerous ways. Keywords can be used that describe pre-set colours such as aqua. RGB values can be used and are noted in the syntax as rgb(253, 30, 56). Hexadecimal can also be used to define the colour, the syntax for this is #ff0000. Opacity can also be controlled, for example the syntax to give 50% opacity using rgb is rgb(253, 30, 56, 0.5).
background-color Text Background Colour This defines the colour of the background to the text for the selected element. This can also be defined in numerous ways. Keywords can be used that describe pre-set colours such as aqua. RGB values can be used and are noted in the syntax as rgb(253, 30, 56). Hexadecimal can also be used to define the colour, the syntax for this is #ff0000. Opacity can also be controlled, for example the syntax to give 50% opacity using rgb is rgb(253, 30, 56, 0.5).
opacity Opacity This defines the opacity to be used for the selected element. This can target numerous things including text and its parameters range from 0 to 1.
background-image Background Image This defines the background image to be used for the selected element. This can be an image, multiple layered images or browsers can render gradients with defined colours, patterns and spacing. When defining an image it always best practice to define a background-color aswell just in case the image fails to load or is unavailable.
!important Important Override The !important override attribute is an attribute that once applied overrides pretty much every other rule. It should almost never be used because of how it ignores even the most specific rules written in the CSS.
border-collapse Border Collapse This defines whether borders in a <table> share borders or have separate borders.

The Box Model

The box model is a set of defining rules on how content takes up space on a page. The linked image shows a visual representation of this.

Tag Name Description
width Content Width This defines the width of the content area.
height Content Height This defines the height of the content area.
padding Content Padding This defines the padding around the content area. The syntax for this can be written in longhand by creating each side's own rule such as padding-top: 2px;, it can be written in shorthand to define each side individually without the need for multiple lines of code such as padding: 0px, 0px, 0px, 0px;, or it can be condensed further to provide a relationship on opposing sides such as padding: 0px, 2px;.
border Content Border This defines the border surrounding the content and padding. This has 3 definable properties that can be noted individually or noted in the shorthand border attribute, these are border-style, border-color and border-width.
margin Content Margin This defines the size of the margins applied to the content. the syntax works in the same way as padding. Margins also make use of the keyword auto, this allows for items to be centred.
box-sizing Box Sizing This defines how a block element's size is determined. There are 2 variables content-box and border-box. A content-box defines the height and width of the content and adds the size of padding and borders on top of the defined height and width. This is the default setting within CSS. A border-box tells the browser to include the padding and border size in the defined height and width, making the content size shrink in relation to the other variables.
min-width, max-width Minimum and Maximum Width This defines the minimum and maximum width of the content area. This helps to keep the content within legible when using flexible layouts.
min-height, max-height Minimum and Maximum Height This defines the minimum and maximum height of the content area. This helps to keep the content within legible when using flexible layouts.
overflow Content Overflow This defines how the browser handles the content of a box when the content is too large for the containing element. When content does overflow it can be visible (the default), hidden (which hides any content that doesn't fit) or scroll which provides a scroll bar to allow the user to scroll down or sideways to view the remaining content.

Display and Positioning

This section looks at tags that define where content takes up space on a page. These tags look at taking the boxes defined by the box model and then positioning them within the page.

Tag Name Description
position Position static - This is the default property for all content. The element stays in the flow of the document, in the position its is located within the markup.
relative - The element stays in the flow of the document however, its position can then be offset and repositioned based on top, right, bottom and/or left attributes. These offsets do not affect the positioning of other elements within the flow of the document. The z-index however, will affect whether it is displayed above or below the other elements in the document.
absolute - The element is removed from the flow of the document and its top, right, bottom and/or left attributes will then determine its position as well as its z-index in relation to its most recent relative positioned parent container element. If an absolute positioned element has no relative positioned ancestors, it uses the document body, and moves along with page scrolling. Elements positioned in this way can overlap other objects if not carefully considered in relation to other margins or padding. This can also be a desired effect.
fixed - The element is removed from the flow of the document and is positioned relative to the document body based on its top, right, bottom and/or left attributes.
sticky - This is a hybrid method of positioning. The element is treated in the same way as a relative element up until the user scrolls past the element's relatively positioned placement, at which point it turns into a fixed method of positioning and moves with the page
z-index Z Index This defines the stack order of an element. The default is 0, however, this property can be adjusted to alter the order elements are displayed on the page in the event of overlapping.
display Display inline - Inline elements use as little spaces as possible and can appear one after the other. They cannot have manually defined height or widths.
block - Block elements take up the entire width of their containing element. They can have manually adjusted heights.
inline-block - Inline blocks take up as little spaces as possible by default but can have both height and widths manually defined. They can be set to appear one after the other within their parent container.
float Float This can tell a block element to be located either all the way to the left, all the way to the right. The default is none.
clear Clear This specifies if an element that is placed after a floated element should be moved down and placed underneath on the page or not.

Signifiers

This section looks at attributes that help to define aspects of the user's interactive experience.

Tag Name Description
cursor Cursor This specifies supplemental states applied to the cursor, usually if hovering over a specific element on the page. These additional states help to provide the user with additional information such as telling them that they can pan on this section of the page or they can click the mouse to zoom in for a closer look.
transition Transition This property allows the CSS to define how interactive elements change states. For example if a button is hovered over or pressed then the transition property will define the timings of the change in states allowing for less jerky visual changes. The syntax is written as transition: transition-property transition-duration transition-timing-function. If there are several items that should be targeted such as the colour as well as the font size then these should be written in the same syntax in series separated by a comma. For example transition: font-size .1s ease-in, color .1s ease-in;

Flexbox

This section looks at tags associated with the flexbox method of display and positioning on a page.

Tag Name Description
flex Block level Flex container This specifies that a div will be displayed as a block level container (the same as its default) however, items inside will now be flex items that can grow and shrink based on defined parameters in the code.
inline-flex Inline Flex container This specifies that a div will be displayed inline. Items inside these divs will also be flex items that can grow and shrink based on defined parameters in the code.
justify-content Justify Content This specifies how flex items will be justified as entire blocks of content within the containing div. The default behaviour is to justify along the horizontal axis (Main axis). As with text, you can justify the content to the left, right and center of the container. There is also functionality to allow for spacing, these are space-around, space-between and space-evenly. Finally you can also justify the content to stretch and fill the container using stretch.
justify-items Justify Items This specifies how flex items will be justified as individual blocks of child content within the grid. The best way to visualise how this differs from justify-content is demonstrated on the MDN web docs page.
align-content Align Content This specifies how flex items will be aligned as entire blocks of content within the containing div. The default behaviour is to align along the vertical axis (Cross axis). This behaves in essentially the same way as the justify-content attribute but along the opposing axis.
align-items Align Items This specifies how flex items will be aligned as individual blocks of child content within the grid. The best way to visualise how this differs from align-content is demonstrated on the MDN web docs page.
flex-grow Flex Grow This specifies the ratio at which flex items will expand along the main axis in relation to the browser window when made larger or smaller. The syntax is written as an integer with no additional units required. For example flex-grow: 1;
flex-shrink Flex Shrink This specifies the ratio at which flex items will shrink along the main axis in relation to the browser window when made larger or smaller. The syntax is written as an integer with no additional units required. For example flex-shrink: 1;
flex-basis Flex Basis This specifies the initial size of a flex item that has the additional flex-grow and/or flex-shrink attributes assigned to it so the browser knows the default size to which the shrink or grow ratios are then applied to.
flex Flex This is the short hand way of writing the flex-grow, flex-shrink and flex-basis parameters in one simplified declartion within the code. One important note to this is that the syntax allows for the flex-shrink section to be missed out in the short hand however you cannot use this declartion to miss the grow and only declare the shrink and basis portions.
flex-wrap Flex Wrap This specifies whether flex items are forced onto a single line or whether they can cascade onto multiple lines. The size of the containing div will dictate the level of wrapping if allowed.
flex-direction Flex Direction This specifies whether flex items are displayed as rows (default behaviour) or columns. The syntax allows for the reversal of both so items are displayed in a row or column with the ordering of the items to be displayed reversed. This is useful for sorting things in ascending or descending order for example.
flex-flow Flex Flow This is the short hand way of writing both the flex-wrap and flex-direction parameters in one simplified declartion within the code.

CSS Grid

This section looks at attributes that create and define grids wthin CSS.

Tag Name Description
grid Grid This attribute is a property of display. This tells the browser to treat the attributed element as a grid. The syntax is display: grid;
grid-template-rows Grid Template Rows This attribute defines the properties of the rows of a CSS grid. The syntax can be written as absolute measurements such as 100px or it can be written as a fraction of the grid's predefined height property, an example of this would be grid-template-rows: 1fr 1fr;. This would give 2 equally sized rows.
grid-template-columns Grid Template Columns This attribute defines the properties of the columns of a CSS grid. The syntax can be written as absolute measurements such as 100px or it can be written as a fraction of the grid's predefined width property, an example of this would be grid-template-columns: 1fr 1fr;. This would give 2 equally sized columns.
grid-template Grid Template This is the short hand way of writing grid-template-rows and grid-template-columns in one declaration. An example of correct syntax is grid-template: 1fr 1fr / 1fr 1fr;. The / within the syntax separates the attributes defining the grid-template-rows from the grid-template-columns.
repeat() Repeat Function This is a function that can be used within the definition of the grid to create simpler code. For example when defining a grid with 3 columns of equal width, you could write grid-template-columns: 100px 100px 100px;. This can be redefined using the repeat function so the code would be grid-template-columns: repeat(3, 100px);.
minmax() MinMax Function This is a function that can define the minimum and maximum values of a grid element. This is particularly useful if an element in the grid is stretching as a result of it being a flex item and you do not wish for it to get smaller or larger than a certain size.
row-gap Row Gap This defines the space between each row item within a grid. It only adds space internally to the grid.
column-gap Column Gap This defines the space between each column item within a grid. As with the row gap, it only adds space internally to the grid.
gap Gap This is the short hand way of writing row-gap and column-gap in one declaration.
grid-row-start Grid Row Start This tells the browser at which row gridline an element should begin within a grid.
grid-row-end Grid Row End This tells the browser on which row gridline an element should end within a grid. This number is always referring to the gridlines of the grid therefore to create an element that spans 2 rows that begins on gridline 1, the grid-row-end declaration should be grid-row-end: 3;.
grid-column-start Grid Column Start This tells the browser at which column gridline an element should begin within a grid.
grid-column-end Grid Column End This tells the browser on which column gridline an element should end within a grid. This number is always referring to the gridlines of the grid therefore to create an element that spans 2 columns that begins on gridline 1, the grid-column-end declaration should be grid-column-end: 3;.
grid-area Grid Area This is the shorthand way of writing grid-row-start, grid-row-end, grid-column-start and grid-column-end. The syntax for this however is quite confusing as it takes in both start points and then both end points all separated by /. For example the correct syntax would be grid-area: grid-row-start / grid-column-start / grid-row-end / grid-column-end;
grid-template-areas Grid Template Areas Grid template areas is a method of laying out a grid based on the names defined in the code. The syntax is best described and visualised on the MDN web docs page.
justify-self Justify Self This works in pretty much the same way as justify-items however this can be used as a more specific override for a particular item if needed.
align-self Align Self This works in pretty much the same way as align-items however this can be used as a more specific override for a particular item if needed.
grid-auto-rows Grid Auto Rows This defines the height of any overflow rows of a grid. If the declaration of the grid in the CSS doesn't account for the entire content, this attribute defines the height of the overflowing rows. This is particularly useful where large amounts of content can be added to the page with functionality built into the site such as a 'display all' feature.
grid-auto-columns Grid Auto Columns This defines the width of any overflow columns of a grid. If the declaration of the grid in the CSS doesn't account for the entire content, this attribute defines the width of the overflowing columns.
grid-auto-flow Grid Auto Flow This defines how the items of a grid are placed if the grid is auto placed rather the explicitly defined. The default is row but this can be altered to column and/or dense. A good visual description of the options for the syntax can be found on the MDN web docs page.

For additional tags and examples see the MDN Website for more info.
