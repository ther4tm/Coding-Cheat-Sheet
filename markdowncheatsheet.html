Command Line Markdown Style Guide

Markdown Cheat Sheet

Below is a brief list of fundamental Markdown tags and a brief description of what they do and any notes on correct syntax.

Fundamentals

This section describes some fundamental Markdown properties.

Tag Name Description
*text* or _text_ Italics An asterisk or an underscore either side of a body of text styles the text in italics.
**text** or __text__ Bold A double asterisk or underscore either side of a body of text styles the text in bold.
# text or
text
====		 Heading 1 A hash or an equals symbol under line styles the text to be a heading 1 element.
## text or
text
----		 Heading 2 A double hash or a minus symbol under line styles the text to be a heading 2 element.
### text Heading 3, 4, 5 or 6 A number of hashes prior to the text style the text to be a heading element that relates to the number of hashes. For example a heading 5 element would require 5 hashesh prior to the text.
[text](http://www.website.com) Link Text contained in square brackets followed by a valid URL will style the text to be a hyperlink that will link to the URL.
![Image](http://www.website.com/image.jpg) Image An exclamation point followed by the keyword Image in square brackets, then followed by a valid URL that links to an image will insert the desired image.
> text
> text		 Blockquote A series of greater than signs will style all lines of text following the greater than symbol as a blockquote.
+ text, * text or - text Unordered List A *, + or - before text will style the text to be a bullet pointed list.
1. text, 2. text Ordered List A number with a period before text will style the text to be a numbered list.
`text` Inline Code Text wrapped in backticks will style the text to be shown as inline code.
```
text
text
```		 Code Block 3 backticks before and after a block of text will style the text to be shown as inline code block.

For additional tags and examples see the MDN Website for more info.
